## 三、数据流

### 用户对话流程

```
1. 用户在 LXfanMallWeb 输入消息
2. 前端通过 SSE 连接 agent-service /api/v1/chat/stream
3. Agent 意图分类（LLM）
4. 根据意图：
   - 工具调用 → 调用 Java 后端 API → 返回结果
   - RAG 检索 → Milvus 向量检索 → Reranker 重排序
   - 直接回复 → LLM 生成
5. SSE 流式返回给前端
6. Token 用量上报到 Java 后端
```

### 商品搜索流程

```
1. 用户输入搜索关键词
2. 前端调用 /search-api/product
3. mall-search 模块查询 Elasticsearch
4. 返回商品列表
```

### 订单超时取消流程

```
1. 用户下单
2. mall-portal 发送延迟消息到 RabbitMQ（30分钟）
3. CancelOrderReceiver 监听队列
4. 30分钟后收到消息
5. 检查订单状态
6. 未支付 → 自动取消，释放库存
```

## 五、开发规范

### 代码风格

- **Java**：遵循原 mall 项目的代码规范
- **TypeScript**：使用 ESLint + Prettier
- **Python**：使用 Ruff 格式化

### Git 提交规范

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
chore: 配置/依赖更新
refactor: 重构
test: 测试
```

### 分支策略

- `main` — 主分支，保持可运行状态
- `dev` — 开发分支（可选）
- `feature/*` — 功能分支（可选）
