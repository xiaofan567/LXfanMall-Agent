# 环境搭建指南

本文档说明如何从零搭建 LXfanMall 的完整开发环境。

## 一、环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| JDK | 8+ | 推荐 JDK 8 或 11 |
| Maven | 3.6+ | 用于编译 Java 后端 |
| Node.js | 18+ | 用于前端开发 |
| Python | 3.11+ | 用于 AI Agent 服务 |
| Docker & Docker Compose | 最新版 | 用于运行基础设施 |
| Git | 最新版 | 版本管理 |

## 二、快速启动（3 步走）

### 第 1 步：启动基础设施

```bash
cd mall/document/docker
docker compose -f docker-compose-env.yml up -d
```

这会启动以下服务：

| 服务 | 端口 | 用途 | 管理界面 |
|------|------|------|---------|
| MySQL | 3307 | 主数据库 | - |
| Redis | 6379 | 缓存/Session/购物车 | - |
| RabbitMQ | 5672/15672 | 消息队列（订单超时取消） | http://localhost:15672 |
| Elasticsearch | 9200 | 商品搜索 | - |
| Kibana | 5601 | ES 可视化管理 | http://localhost:5601 |
| MinIO | 9090/9001 | 对象存储（商品图片） | http://localhost:9001 |
| Milvus | 19530 | 向量数据库（RAG） | - |
| MongoDB | 27017 | 日志存储（可选） | - |

### 第 2 步：初始化数据库

```bash
mysql -h localhost -P 3307 -u root -proot < mall/document/sql/mall.sql
```

### 第 3 步：启动应用服务

详见各服务的启动说明。

## 三、服务详细配置

### MySQL

- **端口**：3307（宿主机） → 3306（容器内）
- **用户名**：root
- **密码**：root
- **数据库名**：mall
- **字符集**：utf8mb4

连接命令：
```bash
mysql -h localhost -P 3307 -u root -proot mall
```

### Redis

- **端口**：6379
- **无密码**
- **持久化**：AOF 模式（appendonly yes）

用途：
- Session 存储（Java 后端）
- 购物车缓存
- RAG 查询缓存（db2）
- 会话记忆（db3）

### RabbitMQ

- **AMQP 端口**：5672
- **管理界面**：15672
- **用户名/密码**：guest/guest

用途：订单超时取消的延迟消息队列

工作流程：
```
下单 → 发送延迟消息到 RabbitMQ（30分钟）
         ↓
30分钟后消费者收到消息
         ↓
检查订单状态 → 未支付则自动取消
```

管理界面：http://localhost:15672

### Elasticsearch

- **HTTP 端口**：9200
- **集群名**：elasticsearch
- **单节点模式**

用途：商品搜索服务（mall-search 模块）

Kibana 管理界面：http://localhost:5601

### MinIO

- **API 端口**：9090（宿主机） → 9000（容器内）
- **控制台**：9001
- **用户名**：minioadmin
- **密码**：minioadmin

用途：商品图片、品牌 Logo 等文件存储

控制台：http://localhost:9001

### Milvus

- **端口**：19530
- **内存限制**：4GB

用途：AI Agent 的 RAG 向量检索

存储内容：
- 文档切片的向量嵌入
- 文档元数据（文件名、策略、切片索引等）

### MongoDB

- **端口**：27017

用途：日志存储（ELK 日志采集，可选）

## 四、常见问题

### 1. Docker 启动失败

```bash
# 查看日志
docker logs mysql
docker logs redis

# 重启服务
docker compose -f docker-compose-env.yml restart
```

### 2. MySQL 连接被拒

检查端口是否正确（3307 而非 3306）：
```bash
mysql -h localhost -P 3307 -u root -proot
```

### 3. Milvus 内存不足

Milvus 默认限制 4GB 内存。如果内存不够，修改 `docker-compose-env.yml`：
```yaml
deploy:
  resources:
    limits:
      memory: 2G  # 改小一点
```

### 4. 端口冲突

如果某个端口被占用，修改 `docker-compose-env.yml` 中的端口映射：
```yaml
ports:
  - 3308:3306  # 改成其他端口
```
