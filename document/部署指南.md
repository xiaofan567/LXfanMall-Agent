# 部署指南

本文档说明如何将 LXfanMall 部署到服务器。

## 一、服务器要求

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB（Milvus 需要 4GB） |
| 硬盘 | 40 GB | 100 GB |
| OS | Ubuntu 20.04+ / CentOS 7+ | Ubuntu 22.04 |

## 二、部署步骤

### 1. 安装 Docker

```bash
# Ubuntu
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo apt install docker-compose-plugin
```

### 2. 克隆项目

```bash
git clone https://github.com/your-username/LXfanMall.git
cd LXfanMall
```

### 3. 启动基础设施

```bash
cd mall/document/docker
docker compose -f docker-compose-env.yml up -d
```

### 4. 初始化数据库

```bash
docker exec -i mysql mysql -uroot -proot < ../../mall/document/sql/mall.sql
```

### 5. 配置环境变量

#### agent-service

```bash
cd agent-service
cp .env.example .env
vim .env
```

填入真实的 API Key：
- `LLM_API_KEY` — DeepSeek API Key
- `EMBEDDING_API_KEY` — 阿里云 DashScope API Key
- `RERANKER_API_KEY` — 同上

#### Java 后端

修改 `mall/mall-admin/src/main/resources/application-prod.yml` 和 `mall-portal` 的对应配置。

### 6. 使用 Docker Compose 部署应用

参考 `mall/document/docker/docker-compose-app.yml`：

```yaml
services:
  mall-portal:
    build: ../../mall/mall-portal
    ports:
      - "8085:8085"
    depends_on:
      - mysql
      - redis
    environment:
      - SPRING_PROFILES_ACTIVE=prod

  mall-admin:
    build: ../../mall/mall-admin
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis

  mall-search:
    build: ../../mall/mall-search
    ports:
      - "8081:8081"
    depends_on:
      - elasticsearch

  agent-service:
    build: ../../agent-service
    ports:
      - "8000:8000"
    env_file:
      - ../../agent-service/.env
    depends_on:
      - milvus
```

### 7. 前端部署

#### 构建

```bash
# 用户端前端
cd LXfanMallWeb
npm run build  # 输出到 dist/

# 管理后台前端
cd mall-admin-web
npm run build  # 输出到 dist/
```

#### Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 用户端前端
    location / {
        root /path/to/LXfanMallWeb/dist;
        try_files $uri $uri/ /index.html;
    }

    # 管理后台
    location /admin {
        alias /path/to/mall-admin-web/dist;
        try_files $uri $uri/ /admin/index.html;
    }

    # Java 后端 API
    location /api/ {
        proxy_pass http://localhost:8085/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /admin-api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 搜索 API
    location /search-api/ {
        proxy_pass http://localhost:8081/search-api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # AI Agent API (SSE)
    location /agent-api/ {
        proxy_pass http://localhost:8000/api/v1/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding off;
        proxy_buffering off;
        proxy_cache off;
    }
}
```

## 三、HTTPS 配置

使用 Let's Encrypt 免费证书：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 四、监控与日志

### 查看服务状态

```bash
docker ps
docker compose -f docker-compose-env.yml ps
```

### 查看日志

```bash
# Java 后端
docker logs mall-portal
docker logs mall-admin

# Agent 服务
docker logs agent-service

# 基础设施
docker logs mysql
docker logs redis
docker logs milvus
```

### 数据备份

```bash
# MySQL 备份
docker exec mysql mysqldump -uroot -proot mall > backup_$(date +%Y%m%d).sql

# Milvus 备份
docker cp milvus:/var/lib/milvus ./milvus_backup_$(date +%Y%m%d)
```

## 五、常见问题

### 1. Milvus 启动失败

检查内存是否足够（至少 4GB）：
```bash
free -h
docker logs milvus
```

### 2. SSE 连接超时

Nginx 配置中需要关闭缓冲：
```nginx
proxy_buffering off;
proxy_cache off;
```

### 3. 跨域问题

确保 Nginx 配置了正确的 `proxy_set_header`。

### 4. 数据库连接失败

检查 MySQL 是否启动，端口是否正确：
```bash
docker exec mysql mysqladmin -uroot -proot status
```
